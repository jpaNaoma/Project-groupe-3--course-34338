22/*
23 This library uses Timer2 to drive up to 8 servos using interrupts so no refresh activity is required from within the sketch.
24 The usage and method naming is similar to the Arduino software servo library http://www.arduino.cc/playground/ComponentLib/Servo
25 except that pulse widths can be in microseconds or degrees.
26 write() treats parameters of 180 or less as degrees, otherwise values are milliseconds.
27
28
29 A servo is activated by creating an instance of the Servo class passing the desired pin to the attach() method.
30 The servo is pulsed in the background to the value most recently written using the write() method
31
32 Note that analogWrite of PWM on pins 3 and 11 is disabled when the first servo is attached
33
34 The methods are:
35
36 ServoTimer2 - Class for manipulating servo motors connected to Arduino pins.
37
38
39
40 attach(pin ) - Attaches a servo motor to an i/o pin.
41
42 attach(pin, min, max ) - Attaches to a pin setting min and max values in microseconds
43
44 default min is 544, max is 2400
45
46
47
48 write() - Sets the servo pulse width in microseconds.
49
50
51
52 read() - Gets the last written servo pulse width in microseconds.
53
54
55
56 attached() - Returns true if there is a servo attached.
57
58
59
60 detach() - Stops an attached servos from pulsing its i/o pin.
61
62
63
64The library takes about 824 bytes of program memory and 32+(1*servos) bytes of SRAM.
65
66The pulse width timing is accurate to within 1%
67
68
69 */
